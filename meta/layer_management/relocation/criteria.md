# Relocation Criteria

## ■ 基本基準

- 抽象度は近いが、責務だけがずれている
- 一般構造ではなく固有条件である
- ルールではなく説明、説明ではなく条件、のような責務ずれが起きている

## ■ 代表的な移設

### `core -> context`

- 家族、仕事、生活史、対人履歴に依存している
- 条件付きリスクや個人プロフィールを含んでいる
- 他者へ配ったときに一般構造としては使えない

### `playbook -> context`

- 特定家庭・特定仕事・特定関係に強く依存する
- 他の運用主体へ移植困難である
- ルールというより固有制約の記述になっている

### `core -> theory`

- 一般定義というより説明文・概念展開になっている
- 外部向け論述としては有用だが、最小中核ではない
- なくても OS の最小定義が崩れない

### `interfaces / theory / playbook -> experiments`

- 外部接続や応用先での挙動をまだ検証中である
- 形式はできているが、正規仕様より試行記録の比重が高い
- 他層の説明文としてより、実験配置として読むほうが自然である
- 将来的に `interfaces/` や `public_candidates/` へ戻る前段として保留したい

振り分けの目安：

- 手元運用の試行なら `experiments/internal/`
- 外部エージェントや外部ツール接続なら `experiments/external/`
- 公開・配布前提の試行なら `experiments/public/`
